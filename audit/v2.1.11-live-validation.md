## 本次重点修复

- 同一证据卡 ID 只能替换同一 `material_id`；跨材料冲突拒绝写盘，防止历史脏数据丢卡。
- 同材料重复卡可以安全去重，JSONL、JSON 和 Markdown 导出保持一致。
- 灵敏度/特异性配对解析避免错绑，并排除工程、分析、传感器等非诊断语境。
- OpenAlex 使用来源原生 ID 作为标识回退，并覆盖 PMID、PMCID、DOI 和 OpenAlex ID 的测试。
- `literature_retmax` 的显式值不再被隐藏的最小值改写。
- 报告打包只读取已有翻译缓存，不在渲染阶段隐式触发网络翻译。
